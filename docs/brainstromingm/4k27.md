# Mindfolio AI 創新替代方案備案

> 狀態：Brainstorming Backup  
> 用途：保留高創新方向，暫不取代目前的 Smart Portfolio Setup 主方案。

## 一、為什麼需要這份備案

目前的 Mindfolio AI 方案完成度高、符合 CMoney 命題，也能在三天內形成完整 Demo；但同類隊伍很可能都會提出「AI 猜持股、AI 建庫、AI 投資助理、AI 焦慮分析」等相近概念。

因此，現行方案的主要競賽風險不是可行性，而是差異化不足：

> 做得完整，不一定代表評審會記得。

這份備案保留一組更具辨識度的上層產品價值，讓專案在不放棄命題適配的前提下，增加可被評審記住的創新亮點。

---

## 二、推薦的雙層產品設計

### 底層：Smart Portfolio Setup

負責直接回答命題：

```text
分析使用者行為
        ↓
提出可能持股
        ↓
使用者快速確認
        ↓
建立最小可用 Portfolio
```

底層價值：

- 降低真實持股輸入門檻
- 用確認取代從零輸入
- 成本與股數可略過
- 區分 AI 推測與使用者確認
- 提升 CMoney 的建庫率與 Portfolio 資料完整度

### 上層：Investment Memory

負責創造差異化與持續價值：

```text
建立 Portfolio
        ↓
用一句話記錄買進原因
        ↓
AI 整理投資假設與預計策略
        ↓
市場或情緒改變時重新比對
        ↓
提醒「策略改變，還是只有情緒改變」
```

上層價值：

- Portfolio 不只保存股票，也保存投資記憶
- 使用者不需要記住數月前的投資初衷
- 市場波動時，以原始理由作為反思基準
- AI 陪伴有具體個人脈絡，不只是通用安慰文字

## 三、核心創新：Investment Memory AI

### 問題

投資人常常不是不知道自己持有哪些股票，而是忘記：

- 何時開始關注或買進
- 當時為什麼買
- 原本預計持有多久
- 哪些條件會讓原始投資理由失效
- 現在想操作，是策略改變還是情緒反應

傳統 Portfolio 只記錄：

```text
股票、股數、成本、損益
```

Investment Memory 額外記錄：

```text
買進原因、投資假設、預計期間、風險條件、後續事件
```

### 一句話定位

> Mindfolio AI 不只幫你記住持有哪些股票，也幫你記住當初為什麼持有。

### 使用範例

建庫完成後，AI 只問一個問題：

```text
你當初為什麼買進台積電？
```

使用者回答：

```text
長期看好 AI 和 CoWoS，預計至少持有一年。
```

AI 整理為：

```json
{
  "stock_id": "2330",
  "thesis": ["AI 長期需求", "CoWoS 成長"],
  "intended_horizon": "at_least_12_months",
  "source": "user_confirmed",
  "recorded_at": "2026-07-12"
}
```

市場波動時，AI 提醒：

```text
你最近查看台積電的頻率是平常的 3.2 倍。

你當初記錄的持有理由是長期看好 AI 與 CoWoS，
預計至少持有一年。現在可以先確認：原本的投資理由
是否真的改變，還是短期價格波動提高了你的焦慮？
```

這不是叫使用者繼續持有，也不是提供買賣建議，而是幫助使用者回顧自己的決策脈絡。

## 四、可組合的創新功能

### A. Why I Bought

每次新增已確認持股時，只問一句：

```text
為什麼買？
```

AI 將自然語言整理成簡短的投資理由與標籤。

優點：

- 輸入門檻極低
- 非常適合 LLM
- 能直接支撐後續陪伴
- 三天 Demo 容易實作

風險：

- 必須保留使用者原文，避免 AI 過度改寫
- AI 整理內容必須讓使用者確認

### B. AI Investment Timeline

將持股與投資理由排列成時間軸：

```text
2024 ETF 穩健配置
  ↓
2025 AI 與半導體
  ↓
2026 機器人與高波動題材
```

AI 可以描述投資偏好如何演變，例如產業集中度或風險偏好是否提高。

適合作為未來擴充，不建議列入三天核心 MVP，因為需要較長時間資料才能產生可信內容。

### C. AI Regret Analyzer

分析使用者在歷史決策後最常出現的後悔模式，例如：

- 過早停利
- 追高後快速賣出
- 恐慌時偏離原始策略
- 賣出後反覆查看同一標的

風險：

- 必須有可靠的歷史交易或使用者確認事件
- 不能把事後股價走勢包裝成當時必然能知道的答案
- 容易滑向投資建議或績效評判

因此不建議成為三天 MVP。

### D. Investment Therapist

以投資心理陪伴方式解釋行為變化：

```text
今天可能不只是市場讓你焦慮，反覆刷新價格本身也可能
持續放大不安。你可以先暫停接收新資訊，回顧原始策略。
```

風險：

- 「Therapist／治療」可能被理解為醫療或心理診斷
- 對外產品命名建議仍使用 Coach、Companion 或 Memory

可保留概念，但不建議正式以「心理醫生」定位。

### E. Strategy Confidence

焦慮分數之外，顯示「原始投資假設目前是否仍有支撐證據」。

但不應輸出看似客觀的 `87% 信心`，除非有清楚且可驗證的計算方式。較安全的呈現方式是：

```text
原始理由：2 項仍待追蹤、1 項已有新變化
```

## 五、三天黑客松建議版本

不要同時實作所有創新功能。最小創新組合為：

1. Smart Portfolio Setup：AI 推測與快速確認。
2. Why I Bought：每檔持股輸入一句買進原因。
3. Investment Memory：AI 整理後由使用者確認。
4. 焦慮事件發生時：同時顯示行為證據與原始投資理由。

### 建議 Demo 流程

```text
畫面一：AI 推測可能持股
        ↓
畫面二：使用者快速確認台積電
        ↓
畫面三：回答「為什麼買？」
        ↓
AI 整理「AI／CoWoS、至少持有一年」
        ↓
畫面四：市場波動後焦慮升高
        ↓
AI 對照原始理由，請使用者判斷策略是否改變
```

### 三天範圍限制

- 只示範一名使用者與一檔核心股票
- 投資記憶只保留一段原文與結構化摘要
- 不做完整時間軸
- 不做真實交易績效與後悔分析
- 不做自動判斷投資理由是否正確
- 不生成買賣建議

## 六、資料需求

### 原本方案需要

- 使用者瀏覽、搜尋、自選股及互動紀錄
- 市場資料
- 使用者確認的 Portfolio

### Investment Memory 額外需要

```text
memory_id
user_id
stock_id
original_text
structured_thesis
intended_horizon
confirmed_at
updated_at
```

其中 `original_text` 必須保存，`structured_thesis` 是 AI 整理結果，兩者不可互相取代。

### 記憶可信度

Investment Memory 只能來自：

- 使用者直接輸入
- 使用者確認過的 AI 摘要

不得只根據瀏覽紀錄，擅自宣稱知道使用者當初的買進理由。

## 七、風險與防線

| 風險 | 防線 |
|---|---|
| 偏離「真實持股」命題 | Smart Portfolio Setup 仍是第一層核心流程 |
| AI 捏造買進理由 | 保存原文，AI 摘要必須由使用者確認 |
| 變相勸使用者續抱 | 只詢問原始理由是否改變，不做持有建議 |
| 三天範圍過大 | 只加入 Why I Bought 與單筆 Investment Memory |
| 沒有歷史交易資料 | 不做 Regret Analyzer 與完整 Timeline |
| 心理醫療風險 | 不使用診斷或治療式產品宣稱 |
| 使用者原始策略本身錯誤 | 不替策略背書，只協助回顧決策脈絡 |

## 九、建議 Pitch

> CMoney 已經很了解市場，卻不一定知道使用者真正持有哪些股票；即使建立了庫存，多數平台也只記得成本與損益，沒有人替投資人記得當初為什麼買。Mindfolio AI 先透過使用者行為提出可能持股，讓使用者用確認取代輸入；建庫後，只需再用一句話留下買進理由。當市場波動與焦慮升高時，AI 不替使用者決定買賣，而是重新呈現當初的投資假設，協助判斷真正改變的是策略、基本條件，還是短期情緒。我們不只建立 Portfolio，也建立每位投資人的 Investment Memory。

---

## 十、啟用這個備案的判斷條件

符合以下條件時，建議升級為主方案：

- 團隊已完成 Smart Portfolio Setup 的 Mock 垂直流程
- 正式資料無法支持高可信的持股模型，但允許使用者確認
- 團隊能在半天內完成一句話原因輸入與確認流程
- Pitch 測試顯示評審對「AI 猜持股」反應普通
- Investment Memory 不會壓縮 Live Demo 穩定化與提案時間

若上述條件未滿足，繼續保留為未來擴充，不影響目前三天 MVP。

## 最終建議

> 不放棄 Smart Portfolio Setup，但將「Why I Bought + Investment Memory」作為唯一創新亮點。底層回答命題，上層創造記憶點。
