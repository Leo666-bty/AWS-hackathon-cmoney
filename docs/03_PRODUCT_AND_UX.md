# Product & UX

## 產品原則

> Zero Prompt、Zero Upload、One Tap。

使用者照常使用籌碼 K 線。AI 不要求切換到聊天頁，也不在第一步要求成本、股數、券商或總資產。

## 主流程

```text
個股頁顯示官方資料
→ Moment Engine 判斷此事件值得出現
→ 洞察卡先說明「發生什麼」與證據
→ 再問股票與使用者的關係
→ 使用者一鍵回饋
→ 更新 Portfolio Context 與後續卡片優先級
```

## 主卡片

### 確認前

```text
法人 × 社群背離

廣達近 20 日法人賣超 60,265 張，
但明確多空貼文中 93.9% 偏多。

廣達與你目前的關係是？
[是我的持股] [只是觀察] [不相關]
```

### 確認後

```text
已確認持股

廣達已加入你的庫存情境。
之後重大籌碼變化會優先出現在今日洞察。
```

## 三個回饋的產品效果

| 回饋 | 系統更新 |
|---|---|
| 是我的持股 | 建立 `holding`，提高重大事件優先級 |
| 只是觀察 | 建立 `watch_only`，保留一般資訊、不用持股語氣 |
| 不相關 | 建立負回饋，降低同類卡片頻率 |

## 推播策略

- 個股頁內嵌卡是主要互動。
- 推播只用於已確認持股的高影響事件。
- 推播先說事件價值，不在通知文字中索取個資。
- 同日最多一則；無重要事件保持安靜。

## 信任設計

個人頁要明確揭露 AI 知道與不知道的資料：

```text
已知道：廣達是否為持股（LEO 親自確認）
不知道：股數、成本、券商、總資產
```

## UX 禁止事項

- 不使用聊天輸入框。
- 不使用截圖上傳。
- 不把候選關係寫成真實持股。
- 不顯示虛假的 AI 準確率或焦慮分數。
- 不用全螢幕強制 onboarding 阻擋看盤。
