# 提案大綱

## 一句話

Mindfolio AI 把 AI 嵌入籌碼 K 線：不用聊天、不用上傳、不用填完整庫存，先在值得注意的市場時刻提供洞察，再讓使用者用一次點擊建立真實持股情境。

## 1. 目標 TA 與情境

LEO 平常會查廣達籌碼、自選股與市場討論，但沒有在 CMoney 建庫。他不會主動打開 ChatGPT 頁面，也不想交出券商與資產資料。

## 2. 不願輸入真實持股的原因與解法

| 阻力 | 解法 |
|---|---|
| 看不到價值前先被要求輸入 | 先提供資料洞察，再詢問關係 |
| 不想聊天或想 Prompt | 沒有輸入框，只提供三個按鈕 |
| 擔心成本、股數與券商外洩 | MVP 只確認是否持有 |
| 建庫與維護太麻煩 | 在個股頁完成一次點擊，不離開原流程 |

## 3. 低門檻與回訪

```text
查看廣達
→ 查看法人、社群與新聞情境
→ AI 顯示法人與社群背離
→ [是我的持股] [只是觀察] [不相關]
→ [有點擔心] [例行查看] [不用提醒]
→ 已確認持股獲得優先事件卡
→ 重大變化時回到原個股頁
```

推播不是主要建庫入口；只有已確認持股且事件影響足夠高時，才用推播帶回 App。

## 4. 成效

北極星：**至少完成一檔使用者確認持股的人數**。

漏斗：個股開啟 → 卡片曝光 → 證據查看 → 一鍵回饋 → 首檔持股確認 → Concern 回覆／情境追問 → Portfolio 查看。

## 5. 技術與資料

- CMoney CSV：個股價量、估值、法人、報酬、產業與社群每日彙總。
- Python：計算 20 日法人、7 日社群比例、背離與卡片優先級。
- Bedrock：把證據轉成結構化 Generative Action Card，不負責算術。
- React：自選股、個股四分頁、情境卡、Portfolio、最小 Dashboard 與 event outbox。
- FastAPI：context、回饋、events batch、metrics。
- PostgreSQL：保存已確認關係、Concern Signal、卡片與互動事件。

## 創新

1. **Reverse Portfolio Onboarding**：先證明價值，再取得確認。
2. **Moment Engine**：不是每次都問，而是選擇最值得打擾的時刻。
3. **Generative Action Card**：AI 產生可操作 UI，不是聊天長文。
4. **Contextual Follow-up**：保留 AI 對話能力，但把空白 Prompt 改成事件當下的三個建議問題。
