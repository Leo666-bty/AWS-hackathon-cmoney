## 一、目標 TA 與使用情境

已在看盤、查資料，但**尚未在任何 App 建立持股庫存**的自主投資人（投資新手到有交易經驗的波段投資人）。
第一次使用時，新手可從分享的「2025 投資人格卡」進場，以五檔股票、大概月份與價格區間，在核心結果前不登入、不上傳、不連券商，約 3 分鐘拿到投資人格與經行情驗證的重建報酬；後續回訪則以年度 Recap 感的回顧，讓有經驗的投資人更新「仍持有」狀態，再換取新的證據型報告。

## 二、用戶不願輸入真實持股的原因與解法

**原因**：不信任（資料去哪）、怕麻煩（股數成本填不完）、沒回報（填了得到什麼）。

**解法：反轉順序——先給價值，再要資料。**
先匿名玩「2025 投資時光機」拿到人格與重建報酬，讓系統先證明「它懂我、算得準」，建立信任之後才問一句：「這幾檔，你還持有嗎？」——只要勾選，不問股數、不問成本、不連券商。兩次獨立同意（認領報告、確認持股）才形成 confirmed holding。

## 三、低門檻輸入方式與持續回訪機制

**輸入只有**：5 檔自選股票 ×（買進月份 + 價格區間 低/中/高 或實際價）× 仍持有/已賣出。模糊記憶即可——後端 Reconstruction Engine 負責驗證行情合理性、處理公司行動（拆股/除權息還原）、擋明顯亂填。

**回訪機制（Portfolio Radar）**：確認持股後，每週基於「真實庫存 × 市場情境」生成優先證據卡與 AI Deep Dive；一鍵回饋（幫我整理證據／例行查看／降低同類提醒）持續校準內容偏好；三個固定問題 chips 取代自由聊天，零 prompt 門檻。

## 四、成效衡量指標

**North Star：Verified Holding Activation** —— 完成重建 → 明確同意 → 至少 1 檔確認持股的人數。

目前尚無真實用戶成效，因此不把假設包裝成既有數字。上線後量測：重建完成率 → 分享卡分享率 → 啟用率（邀請/登入）→ 持股確認率 → 7 日回訪率 → AI Deep Dive 解鎖率與問題 chips CTR，再以 A/B 驗證（詳 `04_GROWTH_AND_SALES_FUNNEL.md`）。

## 五、技術架構與企業資料應用

**架構**（單機 EC2 已上線，Docker Compose：nginx + FastAPI + PostgreSQL）：

```text
React + TS  →  FastAPI /api/v2（14 端點）
                ├─ 確定性重建引擎（mindfolio-core）：所有金融數字
                ├─ 離線 ML 預評分 artifact：市場情境（O(1) 查表，runtime 零 sklearn）
                ├─ Amazon Bedrock（Converse）：evidence-grounded 結構化解讀
                └─ PostgreSQL：報告認領／確認持股／AI cache／互動事件
```

**三層分工是本案的可信度設計**：
1. **確定性引擎**算所有金融數字（價格驗證、公司行動還原、報酬、四軸人格、可信度）——同輸入必同輸出，AI 不算任何一個數字。
2. **離線 ML**（KMeans 市場情境 6 群 + IsolationForest 異常，選模含最大群集 ≤75% gate）以 CMoney 資料預評分 **3,584 個 stock-month 情境**，含 SHA-256 checksum 版本化發布。
3. **Bedrock** 僅將已驗證證據轉成結構化解讀（每句附 evidence ref；schema + 投資建議 guardrail；任何失敗自動 deterministic fallback，Demo 永不中斷）。

**企業資料應用**：個股價量、法人籌碼流量、同學會社群討論（熱度/看漲比例）全數入模；每一份用戶確認的持股回流成 CMoney 最缺的「**承諾層**」資料資產——資料飛輪：更多確認持股 → 更準的個人化證據 → 更高回訪與確認率。

**法遵邊界**：不給買賣建議、目標價、獲利保證；人格是產品標籤非心理診斷；社群資料只作市場情境、不描述個人情緒；Demo 全程合成身份與合成持股，無真實個資。
